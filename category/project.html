Motivation

I built this project to simplify the creation and packaging of django projects. When starting a django project the steps that the followed are usually the same for each project Create virtualenv Download the django and other packages Configure the settings.py Create a .cfg file so that the settings file can read it, the .cfg file usually contains the SECRET_KEY, ALLOWED_HOSTS etc

When its time to deploy its usually involves git pushing and git pulling and then running collectstatic and migrate.

Solution

I decided to check if there would be a way to deploy using the package manager (apt), turns out there is an application called dh-virtualenv, created by the guys at spotify which allows you to package python virtual environments as .deb packages.

So qoqa uses dh_virtualenv to create a .deb package of a django project.

Lets go through qoqa by creating and packaging a django project. Tutorial

Development

qoqa is still under development, but it is at a stage where you can create and django project and its environment and package the project as a .deb file.
